# Contributing

This handbook improves through real-world experience. If you've shipped something with AI tools — or found advice here that was wrong — please contribute. Every fix makes it better for the next person.

## ✅ What we're looking for

- Real-world lessons from using agentic tools in production
- Corrections to outdated or inaccurate advice
- New workflow patterns that others would benefit from
- Clearer explanations of complex concepts

**Good first contributions:** typos, broken links, outdated tool names, or adding a real example to an existing section.

## 🔄 How to contribute

**For small fixes** (typos, broken links, outdated tool info): open a pull request directly.

> [!TIP]
> For larger changes (new sections, restructuring, significant rewrites), open an issue first — it avoids wasted effort if the scope doesn't fit.

1. Fork the repo
2. Create a branch: `git checkout -b improve/section-name`
3. Make your changes
4. Open a PR with a clear description of what changed and why
5. Be open to feedback

## 🚫 What we're not looking for

- Tool-specific tips that don't generalise
- Opinion without experience behind it
- Marketing for specific products

## 💬 Guidelines

- Write in plain English. Prefer plain words over jargon — but assume a senior developer audience; don't over-explain fundamentals.
- Be opinionated. "It depends" is rarely useful in a playbook.
- Keep sections focused. Each guide should cover one topic well, not many topics loosely.
- Prefer concrete examples over abstract descriptions.
- Cite sources for specific claims (statistics, version numbers, dates).
- Update the [CHANGELOG](CHANGELOG.md) when adding notable new content.

## 📋 What belongs here

This handbook covers practices for AI-assisted software development — tools, workflows, prompting, security, and quality. It is intentionally opinionated and practical, not exhaustive.

If you're unsure whether something fits, open an issue and ask.
